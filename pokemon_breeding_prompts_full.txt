POKÉMON ONLINE – ADVANCED IMPLEMENTATION PROMPTS Breeding System + UI
Integration Guide Project: Web Pokémon MMO Stack: HTML, CSS, JavaScript,
Supabase

These prompts are intended for use with an AI coding assistant to
implement a complete breeding system AND integrate it visually into the
existing UI.

IMPORTANT CONTEXT FOR THE AI

The game is a browser-based Pokémon MMO with these characteristics:

• Mostly runs inside a single index.html (SPA architecture) • Uses
HTML + CSS + JavaScript • Uses Supabase for persistence, authentication,
and multiplayer • Existing systems include: - Pokémon capture -
battles - inventory - evolutions - friends - trading - gyms

Pokémon already contain:

species level moves stats IVs (0–31) types owner_id

The new breeding system must NOT:

• modify the IV range • introduce genetic mutations • allow breeding
with other players

The system MUST feel like a natural Pokémon mechanic but optimized for a
browser MMO with idle mechanics.

  ------------------------------------------
  PROMPT 1 — DATABASE STRUCTURE (SUPABASE)
  ------------------------------------------

Design the Supabase database structure required for the breeding system.

Create tables for:

1.  daycare_slots Fields: player_id pokemon_id slot_index (1 or 2)
    deposited_at

2.  eggs Fields: egg_id player_id species parent_a parent_b
    inherited_ivs egg_moves shiny_roll created_at hatch_ready_time

3.  daycare_upgrades Fields: player_id daycare_level egg_capacity
    incubation_speed_bonus

Explain how these tables interact with existing Pokémon tables.

  ----------------------------------
  PROMPT 2 — BREEDING ENGINE LOGIC
  ----------------------------------

Create the JavaScript logic for the breeding system.

The system must:

1.  Detect compatibility using egg groups
2.  Determine egg generation time based on compatibility
3.  Spawn eggs automatically over time (idle mechanic)
4.  Respect egg capacity limits
5.  Generate egg data including:

species inherited IVs egg moves shiny roll hatch timer

Use async functions compatible with Supabase queries.

Provide modular functions such as:

checkCompatibility() generateEgg() calculateInheritance() rollShiny()

  ---------------------------------
  PROMPT 3 — OFFLINE TIMER SYSTEM
  ---------------------------------

Implement an offline timer system so eggs generate while players are
offline.

The system must:

• store timestamps in Supabase • calculate elapsed time when the player
reconnects • generate missing eggs up to capacity

Example logic:

if player left 2 hours ago and egg time = 20 minutes

generate up to capacity eggs automatically.

Provide the algorithm and JS functions.

  PROMPT 4 — EGG INCUBATION ENGINE
  ----------------------------------

Create the egg incubation system.

Each egg has:

hatch_ready_time

Base hatch time:

30 minutes

Actions reduce time:

battle → -2 minutes capture → -3 minutes mission → -5 minutes gym → -10
minutes

Implement functions:

reduceHatchTimer() checkEggHatch() triggerHatchAnimation()
